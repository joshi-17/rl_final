# rl_final — Smart Energy Management RL: Final Evaluation & Comparison

Builds on `rl_mlops`. Adds a fixed baseline agent, head-to-head comparison,
matplotlib plots, SDG alignment, and a full project report.

## How to Run

### Step 1 — Train the RL agent
```bash
cd rl_final
python train.py --config configs/qlearning_v1.yaml --run_id run_1
```

### Step 2 — Run comparison (trains if no policy found, then evaluates + plots)
```bash
python compare.py --episodes 200
```

### Step 3 — Load a pre-trained policy and compare
```bash
python compare.py --load_policy policy_v1.pkl --episodes 200
```

### Step 4 — Run baseline only
```bash
python baseline.py --episodes 200
```

## Analysis

### When RL Performs Better
- **Mixed workloads** — adapts power level to demand instead of always
  using medium power. On low-demand tasks the RL agent saves battery by
  choosing low power; on high-demand tasks it uses high power when battery
  permits.
- **Low-battery situations** — the RL agent has learned to conserve charge
  and avoids draining the last percentage points on low-value tasks.

### Failure Cases
- **Workload spikes** — a sustained burst of high-demand tasks can deplete
  battery faster than the policy anticipates, leading to premature shutoff.
- **Rare states** — (low battery, high demand) may be under-explored if
  training episodes are short; Q-values for such states remain near zero.
- **Distribution shift** — if real-world task probabilities differ from
  training, the learned policy may degrade without online re-training.

### Sensitivity to Workload Spikes
When task demand shifts to consistently "high", the RL agent's conservative
battery management can temporarily hurt scores. The baseline is less volatile
in pure-high-demand bursts, but drains faster overall and performs worse
across mixed workloads.

## SDG Alignment

### SDG 7 — Affordable and Clean Energy
The RL agent reduces **unnecessary energy consumption** on battery-powered
devices by matching power output to actual task demand. Extending battery
life means less frequent charging and a smaller electricity footprint.

### SDG 11 — Sustainable Cities and Communities
Smart power management at the device level reduces the **resource footprint**
of IoT and edge computing deployments. The same power budget can sustain
more devices, enabling denser and more sustainable smart-city infrastructure.
